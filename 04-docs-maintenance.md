# Wisteria Documentation Maintenance Case Study

## Context

The Wisteria Platform documentation set had evolved over time, resulting in:
- inconsistent formatting across pages
- duplicated content in multiple sections
- outdated troubleshooting guidance
- unclear navigation between related topics

The documentation was still functional but required structural improvement to support scalability and usability.

---

## Problem

Users were experiencing difficulty locating relevant guidance, particularly in troubleshooting scenarios. This led to increased reliance on internal support channels for issues that should have been self-service.

---

## Approach

I conducted a structured documentation audit to:
- identify outdated or redundant content
- review consistency across user journeys
- assess gaps in common user workflows

---

## Actions taken

- Rewrote and standardised FAQ-style content
- Improved structure and navigation consistency across pages
- Identified and removed duplicated or outdated guidance
- Aligned updates with simulated release cycles
- Collaborated with technical stakeholders to validate accuracy

---

## Tools used

- Markdown (structured documentation)
- Jira (workflow tracking)
- Confluence-style review processes (simulated)
- AI-assisted drafting tools (Copilot-style workflow)

---

## Outcome

- Improved clarity and consistency across documentation
- Reduced ambiguity in user-facing guidance
- Enhanced usability of troubleshooting content
- Strengthened overall documentation structure and maintainability
